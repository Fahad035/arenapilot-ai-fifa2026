# 📖 Overview

ArenaPilot AI is a next-generation AI-powered stadium operations platform developed for the **Hack2Skill PromptWars Challenge 4**, inspired by the operational requirements of the **FIFA World Cup 2026**.

Modern stadiums generate massive amounts of operational data—from crowd movement and transportation logistics to emergency incidents, accessibility services, and sustainability metrics. ArenaPilot AI transforms this information into actionable intelligence using **Google Gemini AI**, enabling operators to make faster, smarter, and safer decisions.

Rather than functioning as a traditional dashboard, ArenaPilot AI serves as an **AI Operations Copilot**, combining predictive analytics, operational monitoring, and Generative AI recommendations into a unified control center.

---

# 🎯 Problem Statement

Managing a large-scale sporting event involves coordinating thousands of moving parts simultaneously.

Traditional monitoring systems often provide data without meaningful recommendations, forcing operators to manually interpret information during time-critical situations.

ArenaPilot AI addresses this challenge by providing:

- AI-generated operational briefings
- Real-time crowd intelligence
- Risk assessment
- Emergency response recommendations
- Smart route optimization
- Accessibility assistance
- Sustainability insights
- Interactive AI assistant for operational queries

# 🚀 Getting Started

## Prerequisites

Before running the project locally, ensure the following software is installed on your machine.

| Software | Version |
|----------|---------|
| Node.js | 18+ |
| npm | 9+ |
| Git | Latest |
| Google Gemini API Key | Required |

---

# 📥 Clone Repository

```bash
git clone https://github.com/yourusername/ArenaPilot-AI.git

cd ArenaPilot-AI
```

---

# ⚙️ Frontend Setup

Navigate to the frontend directory.

```bash
cd frontend
```

Install dependencies.

```bash
npm install
```

Create the environment file.

```bash
cp .env.example .env
```

Update the environment variables.

```env
VITE_API_URL=http://localhost:5000/api
```

Start the development server.

```bash
npm run dev
```

Frontend will be available at

```
http://localhost:5173
```

---

# ⚙️ Backend Setup

Navigate to backend.

```bash
cd backend
```

Install dependencies.

```bash
npm install
```

Create environment file.

```bash
cp .env.example .env
```

Update

```env
PORT=5000

NODE_ENV=development

GEMINI_API_KEY=YOUR_GEMINI_API_KEY

FRONTEND_URL=http://localhost:5173
```

Run backend.

```bash
npm start
```

or

```bash
npm run dev
```

Backend URL

```
http://localhost:5000
```

```
POST /analyze
```

### Request

```json
{
  "eventName":"Argentina vs Brazil",

  "eventType":"Football Match",

  "attendance":50000,

  "gateA":25,

  "gateB":30,

  "gateC":20,

  "gateD":25,

  "weather":"Storm",

  "accessibilityRequired":false,

  "medicalIncident":false,

  "securityIncident":false,

  "emergencyMode":false
}
```

---

### Success Response

```json
{
   "success":true,
   "analysis":{
      "...":"AI Generated Response"
   }
}
```

### Error Response

```json
{
    "success":false,
    "message":"Invalid scenario data."
}
```
